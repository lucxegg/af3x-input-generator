What is AF3x?

AF3x (AlphaFold 3 Extended) is a modified version of AlphaFold 3 developed at the Kosinski Lab (EMBL) that accepts explicit crosslink restraints as input. Crosslinks are modelled as small-molecule covalent bridges between two residues, giving the network of distance constraints to guide structure prediction.

This tool generates the JSON input files required by AF3x. It handles sequences, crosslinker molecules, disulfide bonds, PTMs, covalent ligand bonds, and batch export for screening.

Quick Start

  1. Add sequences — click + Protein / RNA / DNA / Ligand and paste your sequence or CCD code. Import a PDB/mmCIF or FASTA file for quick setup. Each chain gets an auto-assigned letter (A, B, C…).
  2. Add crosslinks — click + Add crosslinker, choose a preset from the dropdown, then enter chain + residue pairs. Use Pick XL Residues to click directly in the sequence view. One residue position may only appear in one crosslink pair per crosslinker group.
  3. Optional constraints — add disulfide bonds, PTMs, or bonded atom pairs as needed.
  4. Set seeds — enter a number N to generate seeds 1…N, or enter them manually (e.g. 1,2,3,5). More seeds = more diverse model ensemble.
  5. Generate — click Generate JSON. Download the file and pass it to AF3x as the --input_json argument.
  6. Batch export — expand Batch export / Screening to download one JSON per crosslink pair, or random subsets of crosslinks across many jobs.

Supported Crosslinkers

All crosslinkers below are supported by AF3x and available as presets. Symmetric crosslinkers react with both residues identically; asymmetric crosslinkers have distinct chemistry at each end — residue order matters. Dynamic linkers (LINKn, RIGIDn) are generated on the fly with a given chain length; the name in the JSON must include the number (e.g. LINK5).

Note: SDA25A is available in the experimental AF3x branch only. The stable main branch supports SDA, LCSDA, and SDAD but not SDA25A.

Importing Crosslinks from CSV

Click Import from CSV ↑ in the Crosslinks section to load an existing XL-MS result file. Two formats are supported natively:

  • xiNET CSV — output from xiNET / xiSEARCH. Columns: Protein1, PepPos1, PepSeq1, LinkPos1, Protein2, PepPos2, PepSeq2, LinkPos2, …
  • xiVIEW CSV — output from xiVIEW. Columns: Protein1, SeqPos1, Protein2, SeqPos2, …

For other search engines (pLink, MeroX, XlinkX, MS Annika, Scout, MaxQuant/MaxLynx…), start the bundled xkit conversion server in your conda environment and the importer will convert automatically: 

conda activate xkit_env
python xkit_server.py        # starts on http://localhost:5174

After chain mapping you can review, filter, and confirm which crosslinks to import.

Tips & Notes

  • Residue numbering is 1-based — position 1 is the first residue in your input sequence, regardless of PDB numbering.
  • Chain IDs — use the chain ID assigned in the Sequences panel (e.g. A, B). One chain card can represent multiple copies (homodimers) by entering comma-separated IDs: A,B.
  • Asymmetric crosslinkers — residue 1 is the NHS-ester end (reacts with K/S/T/Y/N-term), residue 2 is the diazirine end (any AA). The order matters for how AF3x places the linker.
  • SASD vs Euclidean distance — the max-distance threshold in each crosslinker preset is a solvent-accessible surface distance (SASD). The Euclidean CB–CB distance is always shorter; a pair passing the Euclidean check may still fail SASD if the path goes around an interface.
  • Seeds — AF3x uses model seeds to generate diverse structural models. A single seed produces one model; using 5 seeds gives an ensemble for picking the best-ranked prediction.
  • Version field — the AF3x input format version (typically 2). Change only if your AF3x version requires a different schema.
