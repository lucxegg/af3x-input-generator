Feature Overview

This tool builds AF3x JSON inputs for crosslink-aware AlphaFold 3 structure prediction. Every section below gives you more than one way to enter your data — pick whichever is fastest for you.

Sequences

Type or paste a sequence directly, upload a FASTA file, fetch it automatically by UniProt ID, or import chains and ligands straight out of a PDB/mmCIF structure file.

Crosslinks

Type chain and residue numbers directly, click the pick icon to set positions by clicking residues in the sequence, or import an existing xiNET or xiVIEW CSV result.

Disulfide Bonds

Type cysteine pairs directly, or click the pick icon to set both residues by clicking cysteines in the sequence.

Modifications

Search more than 50 post-translational modifications by name or CCD code, then click a residue to set the position.

MSA

Runs automatically by default. You can disable it entirely for a chain, or supply your own pre-computed .a3m files instead.

Structural Templates

Add custom templates manually — the query/template residue mapping can even be suggested automatically by uploading a PDB or mmCIF file.

Output

Generate, copy, or download the finished JSON, or re-import an existing AF3x JSON file to keep editing it.

What is AF3x?

AF3x is a modified version of AlphaFold 3 developed at the Kosinski Lab (EMBL) that accepts explicit crosslink restraints as input. Crosslinks are modelled as small-molecule covalent bridges between two residues, giving the network of distance constraints to guide structure prediction.

This tool generates the JSON input files required by AF3x. It handles sequences, crosslinker molecules, disulfide bonds, post-translational modifications, custom MSA and structural templates, covalent ligand bonds, a user-provided CCD, and batch export for screening.

Quick Start

  1. Add sequences — click + Protein / RNA / DNA / Ligand and paste your sequence or CCD code. Import a PDB/mmCIF or FASTA file for quick setup. Each chain gets an auto-assigned letter (A, B, C…).
  2. Add crosslinks — click + Add crosslinker, choose a preset from the dropdown, then enter chain + residue pairs. Use Pick XL Residues to click directly in the sequence view. One residue position may only appear in one crosslink pair per crosslinker group.
  3. Optional constraints — add disulfide bonds, modifications (PTMs), or bonded atom pairs as needed. Open Advanced options on a chain for MSA and structural templates.
  4. Set seeds — enter a number N to generate seeds 1…N, or enter them manually (e.g. 1,2,3,5). More seeds = more diverse model ensemble.
  5. Generate — click Generate JSON. Download the file and pass it to AF3x as the --input_json argument.
  6. Batch export — expand Batch export / Screening to download one JSON per crosslink pair, or random subsets of crosslinks across many jobs.

MSA & Structural Templates

Both live under Advanced options on a protein/RNA chain. By default AF3 searches for both automatically — you only need to touch these if you want something different.

  • Without MSA — skip MSA search entirely for this chain (unpairedMsa/pairedMsa set to ""). Faster, but usually lower accuracy.
  • + Add custom MSA — supply your own pre-computed .a3m file(s) (paste, type a cluster path, or upload) instead of letting AF3 search.
  • Without templates — disable template search for this chain (templates: []).
  • + Add template — supply a single-chain PDB or mmCIF structure (content, path, or upload — PDB is converted to mmCIF automatically) plus a 0-based residue mapping between your sequence and the template.
  • Align with sequence & suggest indices — pasting or uploading a structure for a template automatically runs a sequence alignment against it and suggests the query/template index mapping for you to review and apply. This is a convenience built into this tool — AF3x itself has no alignment tool, only the JSON schema, so always check the suggested mapping before trusting it.

Important AF3x rule: if a chain has custom templates, MSA can no longer be left on Auto for that chain — it must also be explicit (either Without MSA or a custom MSA). This tool warns you and blocks JSON generation if you forget; AF3x itself would otherwise reject the whole job with a "templates set only partially" error.

Supported Crosslinkers

All crosslinkers below are supported by AF3x and available as presets. Symmetric crosslinkers react with both residues identically; asymmetric crosslinkers have distinct chemistry at each end — residue order matters. Dynamic linkers (LINKn, RIGIDn) are generated on the fly with a given chain length; the name in the JSON must include the number (e.g. LINK5).

Note: SDA25A is available in the experimental AF3x branch only. The stable main branch supports SDA, LCSDA, and SDAD but not SDA25A.

Importing Crosslinks from CSV

Click Import from CSV ↑ in the Crosslinks section to load an existing XL-MS result file. Two formats are supported natively:

  • xiNET CSV — output from xiNET / xiSEARCH. Columns: Protein1, PepPos1, PepSeq1, LinkPos1, Protein2, PepPos2, PepSeq2, LinkPos2, …
  • xiVIEW CSV — output from xiVIEW. Columns: Protein1, SeqPos1, Protein2, SeqPos2, …

For other search engines (pLink, MeroX, XlinkX, MS Annika, Scout, MaxQuant/MaxLynx…), convert your results to xiNET or xiVIEW CSV format yourself first (e.g. with pyXLMS), then import the converted file here.

After chain mapping you can review, filter, and confirm which crosslinks to import.

Tips & Notes

  • Residue numbering is 1-based — position 1 is the first residue in your input sequence, regardless of PDB numbering.
  • Chain IDs — use the chain ID assigned in the Sequences panel (e.g. A, B). One chain card can represent multiple copies (homodimers) by entering comma-separated IDs: A,B.
  • Asymmetric crosslinkers — residue 1 is the NHS-ester end (reacts with K/S/T/Y/N-term), residue 2 is the diazirine end (any AA). The order matters for how AF3x places the linker.
  • SASD vs Euclidean distance — the max-distance threshold in each crosslinker preset is a solvent-accessible surface distance (SASD). The Euclidean CB–CB distance is always shorter; a pair passing the Euclidean check may still fail SASD if the path goes around an interface.
  • Seeds — AF3x uses model seeds to generate diverse structural models. A single seed produces one model; using 5 seeds gives an ensemble for picking the best-ranked prediction.
  • Version field — the AF3x input format version, fixed at 4 (the current schema version).
  • Description field — a free-text comment per chain (under Advanced options). Purely cosmetic, not used by AF3x for prediction — leave blank unless you want a label in the JSON.
  • User CCD — for custom ligand definitions not in the standard CCD. Provide inline mmCIF content or a file path, not both.
  • Did the template actually get used? AF3x accepting a template (skipping its own search) doesn't guarantee the predicted structure closely matches it — templates are a soft signal, not a hard constraint. To check: (1) look for "Skipping MSA and template search… because it already has templates" in the AF3x log, and (2) for a stronger check, run the same job again with Without templates and compare pLDDT/confidence between the two runs, or structurally align the predicted model against the template mmCIF.
